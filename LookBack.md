# Day1 - 리눅스 서버 관리자

## Q. 원래 명령어만 이렇게 배우는 건가?

> 그러면 서버 관리자는 왜 필요한 거지?
> 그냥 구글에서 검색해서 확인하면 되는 거 아닌가?

### A.

서버 관리자는 **명령어를 외우는 사람이 아니라, 서버의 상태를 판단하는 사람**입니다.

명령어는 문서나 검색을 통해 언제든 찾아볼 수 있습니다. 하지만 중요한 것은 다음 두 가지입니다.

* 지금 상황에서 **무엇을 확인해야 하는지**
* 명령어 결과가 **정상인지 비정상인지 판단하는 능력**

실무에서는 다음 과정이 반복됩니다.

1. 필요한 명령어를 검색하거나 힌트를 얻는다.
2. 명령어를 실행한다.
3. 결과를 해석하고 원인을 판단한다.
4. 비슷한 상황이 다시 오면 더 빠르게 문제를 해결한다.

즉, 경험이 쌓일수록 명령어 자체보다 **판단하는 감각**이 늘어나게 됩니다.

---

## 느낀 점

처음에는 단순히 명령어를 입력하고 결과를 확인하는 과정이라 재미를 크게 느끼지는 못했습니다.

하지만 Day3~Day4에서 직접 서버를 구축하면서 Day1에서 배운 내용들이 실제로 연결되기 시작했습니다.

**기초를 배우는 시간이었다는 점에서 의미가 있었습니다.**

다만 Linux CLI는 아직도 UI가 익숙하지 않아 적응이 필요하다고 느꼈습니다.

# Day2 - 네트워크

## 느낀 점

학교에서 배운 CS 지식은 시험이 끝나면 대부분 잊어버렸다는 것을 느꼈습니다.

이번 실습을 하면서 운영체제와 네트워크는 따로 시간을 내어 이해 중심으로 공부해야겠다는 생각이 들었습니다.

또한 보안 분야를 공부할 때보다 부담감이 적었습니다.

보안은 공격자의 입장에서 허점을 찾아야 한다는 압박감 때문에 어렵게 느껴졌지만,

네트워크는 구조를 이해하고 명령어를 통해 하나씩 확인해 나가는 과정이라 **노력하면 충분히 따라갈 수 있겠다**는 느낌을 받았습니다.

### 멘토 피드백

보안은 공격자의 사고방식으로 예외 상황과 허점을 찾아내는 **창의적·직관적 사고**의 비중이 큰 편입니다.

반면 인프라는 **왜 문제가 발생했는지 구조적으로 하나씩 확인하며 해결하는 절차적 사고**의 비중이 더 큽니다.

# Day3 - 웹 서버 (Nginx)

## 느낀 점

3일 차가 되니 터미널에서 명령어를 입력하고 결과를 해석하는 과정이 조금씩 익숙해지기 시작했습니다.

아직 완벽하지는 않지만,

예전보다 로그를 읽고 "왜 이런 결과가 나왔는지" 생각하는 습관이 생기기 시작했습니다.

또한 명령어를 단순히 외우기보다,

**처리 과정과 문제 해결 흐름을 반복해서 복습해야겠다**는 생각이 들었습니다.

마지막 날에는 스스로 문제를 만들어 시험을 보며 복습하면 더 오래 기억에 남을 것 같습니다.

### 멘토 피드백

명령어 암기 테스트보다는

> **"이 로그를 보고 원인을 설명해 보세요."**

처럼 결과를 해석하는 문제를 만드는 것이 실제 실무 감각을 기르는 데 훨씬 도움이 됩니다.

또한 보안 분야보다,

원인을 찾아 하나씩 해결하는 과정이 저와 더 잘 맞는다는 느낌을 받았습니다.

# Day4 - Docker

## 느낀 점

Docker는 현업에서 많이 사용한다는 이야기를 정말 많이 들었습니다.

하지만 막상 직접 사용해 보니 생각보다 어렵게 느껴졌습니다.

그래서 오히려 시작하기가 조금 두렵기도 했습니다.

다만 현업에서는 거의 필수 기술에 가까운 만큼,

피할 수 없는 기술이라는 생각이 들었고 꾸준히 공부해야겠다고 느꼈습니다.

또 하나 느낀 점은,

전날 네트워크를 공부한 덕분에 Docker의 포트 포워딩이나 Reverse Proxy 같은 개념을 조금은 이해할 수 있었다는 것입니다.

여전히 어렵지만,

이전보다 "왜 이런 구조인지"를 조금씩 연결해서 이해하게 되었습니다.

### 멘토 피드백

지금 Docker가 어려운 이유는 Docker 자체가 특별히 어려운 기술이기 때문이 아니라,

운영체제, 프로세스, 네트워크 등 여러 기초 개념이 함께 사용되는 기술이기 때문입니다.

즉, 지금 어렵게 느껴지는 것은 그동안 배운 내용들이 실제로 하나의 시스템으로 연결되기 시작했다는 신호이기도 합니다.
