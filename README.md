# 🚀 대학교 2학년을 위한 인프라 엔지니어 직무 체험 레포지토리 (1주일 커리큘럼)

본 저장소는 **인프라 엔지니어(Infrastructure Engineer / DevOps)**의 핵심 업무 흐름을 1주일 동안 압축적으로 경험해보는 프로젝트입니다. 

단순히 기술 명령어를 외우는 것을 넘어, 현업에서 매일 발생하는 **'장애 티켓 대응 -> 원인 격리 -> 트러블슈팅 -> 문서화'**의 전 과정을 시나리오 기반으로 체험하게 됩니다.

---

## 💡 실습 기조
> **"기술은 얕고 넓게, 업무 흐름은 현업과 똑같이"**
인프라 엔지니어의 핵심 가치는 모든 명령어를 외우는 것이 아니라, **계층적으로 문제를 좁혀가는 사고방식**과 **누구나 재현할 수 있도록 만드는 문서화 능력**에 있습니다.

---

## 📅 1주일 커리큘럼 요약

| 폴더명 | 담당 직무 | 핵심 실습 내용 | 권장 소요 시간 |
| :--- | :--- | :--- | :--- |
| **day1/** | **Linux 서버 관리자** | 서버 리소스(CPU/Mem/Disk) 점검, `systemd` 서비스 상태 확인 및 로그 진단 | 3~4시간 |
| **day2/** | **네트워크 담당자** | SSH 키 기반 인증, 포트 오픈 점검, DNS 조회 및 네트워크 장애 원인 격리 | 3~4시간 |
| **day3/** | **웹 서버 운영자** | Nginx 설치, 정적 페이지 배포, Reverse Proxy 구성, 502 에러 로그 분석 | 3~5시간 |
| **day4/** | **Docker 운영 담당자** | 이미지 빌드, 컨테이너 격리 네트워크 연동, 볼륨을 이용한 데이터 영속성 확보 | 3~4시간 |
| **day5/** | **클라우드 운영자** | AWS EC2/VPC 인프라 구축, Security Group 제어, 클라우드 비용 체계 감각 익히기 | 4~5시간 |
| **day6/** | **통합 배포 & 문서화** | AWS 환경에 Nginx + Docker 컴포넌트 통합 배포, 운영 절차서(Runbook) 작성 | 4~5시간 |
| **day7/** | **회고 및 진로 판단** | 직무 성향 자가 진단 설문, 인프라 커리어 패스 및 넥스트 스텝 수립 | 1~2시간 |
