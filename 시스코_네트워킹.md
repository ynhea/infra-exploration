# Day 5 - IP / 서브넷 마스크 / 라우팅 테이블

## 1. IP

- 32자리 이진수(Binary)로 구성되어 있음.
- 보통 십진수로 표현하며, 점(`.`) 3개를 사용하여 4개의 자리(옥텟)로 표시함.
- IP 주소는 네트워크(Network) 부분과 호스트(Host) 부분으로 나누어짐.
- 네트워크 부분은 같은 네트워크에서는 모두 같아야 함.
- 호스트 부분은 같은 네트워크에서는 모두 달라야 함.
- 클래스(Class)를 통해 네트워크 부분과 호스트 부분을 구분함.
- Class A : 맨 앞 이진수가 `0`이고, 십진수 범위는 `1 ~ 126`.
- Class B : 맨 앞 이진수가 `10`이고, 십진수 범위는 `128 ~ 191`.
- Class C : 맨 앞 이진수가 `110`이고, 십진수 범위는 `192 ~ 223`.

## 2. 서브넷 마스크

- 서브넷의 목적은 큰 네트워크를 작은 네트워크로 나누기 위함.
- IP 주소를 효율적으로 낭비 없이 사용하기 위함.
- 적절한 IP 주소를 배정하기 위함.
- 서브넷 마스크는 서브넷을 만들 때 사용하는 값.
- 모든 IP 주소에는 디폴트(Default) 서브넷 마스크가 존재함.
- 이진수에서 `1`은 네트워크 부분을 의미함.
- 이진수에서 `0`은 호스트 부분을 의미함.
- IP 주소에서 호스트 부분이 모두 `0`이면 네트워크 주소(Network Address)를 의미함.
- IP 주소에서 호스트 부분이 모두 `1`이면 브로드캐스트 주소(Broadcast Address)를 의미함.
- 서브넷도 하나의 네트워크이므로 다른 서브넷과는 라우터를 통해서만 통신 가능함.
- 서브넷 마스크는 이진수에서 `1`들 사이에 `0`이 오면 안 됨.

## 3. 라우팅 테이블

- Router는 목적지까지 가기 위해 어느 인터페이스로 전송해야 하는지를 라우팅 테이블에 저장함.
- 라우팅 테이블은 RAM에 저장됨.
- 전원이 꺼지면 라우팅 테이블은 모두 삭제됨.
- 전원을 켜면 수초~수분 내에 다시 생성됨.
- 라우팅 테이블은 사용하는 Router의 라우팅 프로토콜에 따라 달라짐.
