# Day 5 - IP / 서브넷 마스크 / 라우팅 테이블

## 1. IP (Internet Protocol)

- 32자리 이진수(Binary)로 구성되어 있음
- 보통 **십진수(Dotted Decimal Notation)** 로 표현하며, 점(`.`) 3개를 사용하여 4개의 옥텟(Octet)으로 표시함.
  - 예) `192.168.0.1`
- IP 주소는 **네트워크(Network) 부분**과 **호스트(Host) 부분**으로 나누어짐.

### 네트워크 부분
- 같은 네트워크에 속한 장비들은 **모두 동일해야 함**

### 호스트 부분
- 같은 네트워크 내에서는 **모두 달라야 함**

### 클래스(Class)

네트워크 부분과 호스트 부분은 **클래스(Class)** 를 통해 구분됨.

| 클래스 | 맨 앞 이진수 | 십진수 범위 |
|---------|-------------|------------|
| Class A | `0` | 1 ~ 126 |
| Class B | `10` | 128 ~ 191 |
| Class C | `110` | 192 ~ 223 |

# 2. 서브넷 마스크 (Subnet Mask)

## 서브넷(Subnet)의 목적

- 커다란 네트워크를 잘게 나누기 위해 필요함.
- IP 주소를 보다 효율적으로, 낭비 없이 사용하기 위함.
- 적절한 주소를 배정하기 위함.

## 서브넷 마스크

- 서브넷을 만들 때 사용하는 값.
- 모든 IP 주소에는 **디폴트(Default) 서브넷 마스크**가 존재함.

### 이진수 기준

- `1` → 네트워크(Network) 부분
- `0` → 호스트(Host) 부분

### 특별한 IP 주소

- 호스트 부분이 모두 `0`
  - → **그 네트워크 자체(Network Address)**

- 호스트 부분이 모두 `1`
  - → **브로드캐스트 주소(Broadcast Address)**

### 서브넷 통신

- 서브넷도 하나의 네트워크이므로,
- **다른 서브넷과의 통신은 라우터(Router)를 통해서만 가능함.**

### 서브넷 마스크 규칙

- 이진수로 표현했을 때
- **`1`들 사이에 `0`이 오면 안 됨.**

예시

```
11111111.11111111.11110000.00000000   (O)
11111111.11101111.11110000.00000000   (X)
```

# 3. 라우팅 테이블 (Routing Table)

- Router는 목적지까지 패킷을 보내기 위해,
- **어느 인터페이스(Interface)로 보내야 하는지**를 라우팅 테이블에 저장함.

### 특징

- 라우팅 테이블은 **RAM**에 저장됨.
- 따라서 전원이 꺼지면 모두 삭제됨.
- 전원을 다시 켜면 **수초~수분 내에 자동으로 다시 생성됨.**

### 참고

- 라우팅 테이블의 생성 방식은
- **사용하는 Router의 라우팅 프로토콜에 따라 달라짐.**
